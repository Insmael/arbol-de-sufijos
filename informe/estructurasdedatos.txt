Para la construcción de la estructura de datos del árbol de sufijos, se decidió primero crear la estructura Trie. Luego, a partir del Trie crear un árbol Patricia, y por último, a partir del árbol patricia, crear el árbol de sufijos.

árboles trie:
los árboles trie tienen dos tipos de nodos, los internos y las hojas.
todos los strings que contiene el trie terminan en $, y lo que diferencia a un nodo hoja de un nodo interno es último caracter que llevo hasta el nodo, siendo $ para un nodo hoja, y cualquier otro para un nodo interno, incluyendo el carácter '' (vacío) para la raíz del árbol.
Modelamos este ordenamiento con diferentes clases. Una clase para las hojas y una clase para los nodos internos.
Para el caso particular de que el árbol este vacío, se tiene una clase especial, que a todas las búsquedas devuelve falso.
la construcción del árbol trie, entonces, consiste en ir insertando strings en la estructura, e ir extendiendola de ser necesario.
hay dos casos respecto a la inserción: que se tenga que modificar el árbol, o que no se tenga que modificar el árbol. Si no se tiene que modificar el árbol, es debido a que el string que se intenta insertar ya está en aquel, y corresponde a alcanzar un nodo hoja mientras se inserta, pues el último carácter de todos los string es $, que es el camino que nos lleva a los nodos hoja. Considerando esto y que el carácter $ solo se puede ubicar al final del string y en ningúna otra posición, damos por terminada la inserción.
En cambio, si se tiene que modificar el árbol, tenemos dos casos, que tengamos que extender un nodo interno, o que tengamos que extender el nodo especial, que representa un árbol vacío. El extender un arbol vacío consiste en reemplazar el nodo especial por un nodo interno, y proceder con la inserción sobre el nodo raíz, como si el trie siempre lo hubiese tenido de raiz. Si es en cambio un nodo interno el que tenemos que extender, entonces debemos crear un nuevo hijo para el nodo con el siguiente carácter que toque leer del string. dependiendo de este carácter, o creamos una nueva hoja, caso en el que la inserción termina, o tenemos que crear un nuevo nodo interno, caso en el que debemos extender el nuevo nodo interno como ya se explico.
